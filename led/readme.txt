led实现的四种方式：
1、简单驱动+应用方式：驱动中实现open read write 等函数，并映射物理地址。应用实现open write即可。
2、驱动+简单设备树+应用：物理地址放在设备树中指定，驱动获取设备树信息，并映射。
3、驱动+设备树+应用：设备树中通过princtl和GPIO子系统，指定GPIO的引脚编号。驱动中获取编号即可。
4、上一版本的升级版本，已知引脚编号，直接在应用层导出节点属性，使用。

总结：最直接方式 1。
		 最优的方式 3 4。

对应设备树文件添加如下内容：
根节点下添加：
	zynq_led0 {
	#address-cells = <1>;
	#size-cells = <1>;
	compatible = "zynq,leddrv";
	status = "okay";
	 reg =< 	0xF800012C 		0X04 	/* GPIO时钟使能 */
					0xE000A288 		0X04 	/* 输出使能 */
					0xE000A284 		0X04 	/* 方向控制 */
					0xE000A010 		0X04 >;/* 数据控制寄存器 */
	};
	
	zynq_led1{
		led-gpio = <&gpio0 54 0x00>;	/*led的另一种实现方式*/
	};
